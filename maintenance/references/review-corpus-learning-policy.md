# Historical Plan Learning Policy

Use historical tracking plans to improve analyst judgement without copying
client details or obsolete schemas.

## Privacy Boundary

Do not copy client names, domains, URLs, screenshots, IDs, emails, phone
numbers, container IDs, measurement IDs, test notes, or raw workbook rows into
the reusable skill.

## Platform Boundary

- Learn GA4 event grouping, business scenarios, parameter rules, and workbook
  patterns only after checking current official Google documentation.
- From Universal Analytics or another analytics platform, learn only business
  intent, journey design, workbook readability, and useful analyst questions.
- Never reuse another platform's event names, parameters, property names,
  payloads, custom-dimension numbering, or ecommerce structure in a GA4 plan.

## Useful Lessons

- concise navigation and version history;
- journey-grouped event matrices;
- parameter dictionaries with display names and controlled values;
- explicit data availability and implementation ownership;
- business, conversion, diagnostic, and context roles;
- screenshot evidence useful to analysts and developers;
- deliberate exclusions for noisy, duplicate, unavailable, or sensitive data.

## Reject

- one custom event per button, card, banner, or link;
- incomplete GA4 ecommerce events;
- fields copied from Universal Analytics;
- direct identifiers, free text, or sensitive user input;
- client-specific screenshots or implementation defects;
- dense layouts that obscure the Event Matrix.

## Promotion Gate

Promote a lesson only when it is current, generic, privacy-safe, useful to a
human analyst or developer, maintainable, and compatible with official GA4
semantics.
