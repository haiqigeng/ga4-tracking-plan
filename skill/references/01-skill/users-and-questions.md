# Users And Questions

## Primary Users

- Web analysts and analytics consultants creating or reviewing GA4 plans.
- Analytics engineers and tracking-plan owners governing measurement design.
- Developers implementing GTM and dataLayer requirements.

Marketing, ecommerce, product, content, and media teams are secondary readers
when they need to understand the measured business outcomes.

## Questions Resolved

- What should be measured for this website, page, or journey, and why?
- Which GA4 automatic, enhanced-measurement, recommended, or ecommerce event
  fits each business action?
- When is a custom event justified?
- Which parameters are needed, how should values be governed, and where should
  each value come from?
- Which parameters require development, backend support, or owner confirmation?
- Which events belong to the same journey and how should they be grouped?
- What should remain untracked because it is noisy, duplicate, unavailable,
  sensitive, or not actionable?
- Can the plan support future pages, markets, and journey variants without a
  redesign?
